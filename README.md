# Holberton School - Sorting Algorithms & Big O

This repository contains my C solutions for the **Holberton School - Sorting
Algorithms** project. It focuses on classic and advanced sorting algorithms,
Big O analysis, and working with both arrays and doubly linked lists.

## Requirements

- OS: Ubuntu 20.04 LTS
- Compiler: `gcc`
- Flags: `-Wall -Werror -Wextra -pedantic -std=gnu89`
- Style: Betty
- No global variables
- Max 5 functions per file
- All files end with a new line
- No standard library functions (e.g., `printf`, `puts`) in your code
- Header: `sort.h` (include-guarded, contains all prototypes)
- Lists/arrays of size `< 2` are considered already sorted

## Data Structures & Helpers

- Doubly linked list type: `listint_t`
- Helper functions provided by the checker: `print_array`, `print_list` (prototypes included in `sort.h`)

## Tasks

### Mandatory (0-3)
- **0. Bubble sort** - sort an array using bubble sort
- **1. Insertion sort** - sort a doubly linked list using insertion sort
- **2. Selection sort** - sort an array using selection sort
- **3. Quick sort** - sort an array using quick sort (Lomuto partition)
- **Big O** - `0-O`, `1-O`, `2-O`, `3-O`

### Advanced (100-1000)
- **100. Shell sort** - sort an array using Shell sort (Knuth sequence)
- **101. Cocktail sort** - sort a doubly linked list using cocktail shaker sort
- **102. Counting sort** - sort an array of integers using counting sort
- **103. Merge sort** - sort an array using merge sort
- **104. Heap sort** - sort an array using heap sort
- **105. Radix sort** - sort an array using LSD radix sort
- **106. Bitonic sort** - sort an array using bitonic sort
- **107. Quick sort (Hoare)** - sort an array using Hoare partition scheme
- **1000. Sort deck** - sort a deck of cards (custom data structure)
- **Big O** - `101-O`, `102-O`, `103-O`, `104-O`, `106-O`, `107-O`
